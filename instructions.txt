
You are my senior full-stack hackathon engineer. Build a mobile-first web app called Coffee Passport for a Google hackathon.

Core concept

Coffee Passport is an AI-powered cultural coffee discovery experience. The user:

uploads a selfie or photo
gets a playful coffee recommendation based on their vibe/style, not biometric judgment
unlocks a digital coffee passport stamp
explores the coffee’s origin, tasting notes, and story
chats with the coffee as if the coffee has a distinct personality
optionally sees nearby cafés where they might try similar coffee in real life

The app should feel creative, polished, demo-friendly, and realistic to finish in a hackathon.

Critical constraints

Design and implement with these constraints:

This is for a hackathon, so prioritize clarity, demo quality, reliability, and speed of development
We have only $5 in Google Cloud credits
Minimize paid API usage aggressively
Avoid overengineering
Use a small curated local dataset of coffees instead of a large real database
Make the app work even if external APIs are unavailable
Build as a mobile-first web app / PWA
Do not build a native mobile app
Do not build unnecessary auth or complex multi-user systems unless trivial
Keep architecture simple enough for 1–2 developers to understand quickly

Main features

Implement these features:

1. Coffee Passport

Each coffee should have:

2. Photo-based vibe recommendation

The user uploads a photo and gets a coffee recommendation.
Important:

this must be framed as a playful vibe/style-based recommendation
it must not infer protected traits, identity, attractiveness, or anything creepy
it must not claim scientific accuracy

The pipeline should be:

accept uploaded image
analyze image into soft vibe tags such as:
bold
cozy
adventurous
calm
minimal
vibrant
elegant
playful
map vibe tags to coffee profiles in the local dataset
return one main recommendation and optionally 2 alternates
provide a short, friendly explanation of “why this coffee”


3. Talk with the coffee

Users can chat with a recommended coffee.
Each coffee should have a distinct conversational identity based on:

origin
roast
flavor notes
personality traits
story snippet

The coffee should:

speak in first person as the coffee
be fun and memorable
stay grounded in the provided metadata
avoid hallucinating factual origin details not in the dataset
answer questions like:
what do you pair well with?
when are you best to drink?
why do you taste this way?
what makes you different from other coffees?

Implementation note:

use straightforward prompting and structured context
keep the system easy to swap later if we want to experiment with steering
4. Nearby cafés via Google Places API

After a user gets a coffee recommendation, optionally show nearby cafés or coffee shops where they might try similar coffee.
Use Google Places API in a lightweight way.
Requirements:

user can input a location or allow browser geolocation
show nearby coffee shops / cafés
show name, rating, address, and maybe photo if easy
keep requests minimal to protect credits
degrade gracefully if Places API is unavailable

Frame this as:
“Try this coffee vibe near you.”
